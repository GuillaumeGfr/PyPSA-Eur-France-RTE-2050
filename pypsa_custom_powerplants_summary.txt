Modified PyPSA-Eur Workflow: Scenario-Specific Powerplants (RTE 2050)

Overview:
This setup enables fixed-capacity simulations for different scenarios by loading custom CSVs for both conventional and renewable powerplants. It bypasses default powerplantmatching logic and enforces scenario-specific generator capacities.

Key Modifications:

1. build_powerplants.py
- Loads only from a scenario-specific CSV defined as input.RTE_powerplants in the smk file
- Skips powerplantmatching and OPSD database logic.


2. add_electricity.py
- Adds renewable capacities using assign_custom_renewables_from_csv().
- Reads from a CSV with columns: scenario, region, carrier, capacity.
- Distributes capacity by region using p_nom_max as weights (how pypsa usually works)
- Sets p_nom and p_nom_min and disables p_nom_extendable.

3. build_electricity.smk (Snakemake Rule)
- Rule build_powerplants: uses input.RTE_powerplants = data/powerplants_rte/powerplants_{config['run']['name']}.csv. for conventional, nuke and hydro, with run name being one of the scenarios
- Rule add_electricity: uses input rte_regions_path="data/RTE_regions.csv", one file with renewables per region per scenario


4. config.yaml
Example:
run: 
   name: "M23"

Make sure to name runs after one of the RTE scenarios, or modify the rule input to account for a different run name. The edits do not make use of the scenarios function built into PyPSA


Input Files Required:
- data/powerplants_rte/powerplants_<scenario>.csv (conventional plants)
- data/RTE_regions.csv (regional renewable capacities)

Output:
- resources/powerplants_s_151.csv — processed powerplants
- networks/base_s_151_elec.nc — network with generators and storage
